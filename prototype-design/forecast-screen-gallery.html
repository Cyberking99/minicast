Forecast — Prediction Pool Frontend

Complete production-ready design for a USDC-based prediction pool app with Venice AI oracle resolution. 6 screens × light/dark/mobile variants. Full design system below.

Feed
Feed with pool cards, stake bars, categories
1. Home / Feed
Light Desktop
Key decision: Three feed sections (Trending, Closing Soon, Newly Created) with a unified card design — the stake bar is the visual anchor, not text. This lets users scan probability splits instantly.

2. Pool Detail
Light Desktop
Key decision: Sticky sidebar stake panel vs bottom-sheet on mobile. The inline stake input shows live odds feedback as the user types — crucial for financial decisions. The AI analysis is a modal overlay, preserving the pool context.

3. AI Analysis Report
Light Desktop
Key decision: Blurred preview + paywall overlay before unlock simulates the x402 micropayment flow. The confidence arc visualization (not just a number) makes the AI's certainty tangible. Venice AI attribution builds credibility.

Create
Focused form with preview step
4. Create Pool
Light Desktop
Key decision: Composing-a-post minimalism instead of an intimidating multi-tab form. Dynamic option rows (min 2, max 10), date presets for deadlines, and a Preview step before submission — reduces cognitive load for first-time creators.

Portfolio
Positions, history, P&L summary
5. My Portfolio
Light Desktop
Key decision: Summary strip at top gives immediate P&L clarity — the most important financial metric. Position cards with pool-share mini-bar show at a glance whether your option is winning. Color coding (green/red) maps to gain/loss.

6. Settlement / Payout
Light Desktop
Key decision: No confetti — a clean, confident checkmark + the dollar amount in large type communicates the win with dignity. The oracle verdict is displayed prominently as a feature, not a footnote. Transaction hash copy button enables verification.
